커밋을 되돌릴 때 사용
reset --hard

git reset --hard <커밋체크섬> : 해당 커밋으로 되돌리고 싶을 때 사용, 워킹트리의 내용도 함께 초기화
git reset -- mixed <커밋체크섬> : 옵션을 생략할 경우 기본. 워킹트리의 파일은 남아 있고 스테이지만 초기화.
git reset --soft <커밋체크섬> : 커밋은 돌아갔지만 스테이지와 워킹트리의 내용은 변경되지 않는다.

reflog : 커밋 또는 브랜치의 변경사항을 기록해 놓은 로그
git reflog [-n숫자] 갯수를 제한해서 볼 수 있다.
